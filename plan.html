🗓️ ПОШАГОВЫЙ ПЛАН: QA AUTOMATION (PLAYWRIGHT + TYPESCRIPT + OWASP)

МЕСЯЦ 1: Фундамент (JS/TS, API и Git)

Неделя 1: Реанимация JavaScript (База для тестов)

  • День 1: Настройка среды (Node.js, VS Code). Запуск JS через консоль (node index.js). Переменные let и const.
  • День 2: Управляющие конструкции. Условия (if/else), циклы (for/while) и логические операторы сравнения (===, &&, ||).
  • День 3: Функции. Современный синтаксис стрелочных функций (const func = () => {}), передача аргументов и возврат значений (return).
  • День 4: Объекты {} и Массивы []. Структурированное хранение данных, индексы элементов, обращение к свойствам через точку.
  • День 5: Методы массивов. Автоматический перебор и фильтрация списков через встроенные методы .forEach() и .filter().
  • День 6: Асинхронность. Понимание Promise, синтаксис async / await, управление ожиданиями ответов от сайта и сервера.
  • День 7: Финал недели. Сборка комплексного асинхронного скрипта фильтрации и обработки данных. Подведение итогов.

Неделя 2: Переход на TypeScript и база Git

  • День 8: Знакомство с TypeScript (TS). Установка компилятора (tsc), базовая статическая типизация данных (string, number, boolean). Строгий режим "use strict".
  • День 9: Типизация функций. Строгое описание типов входных аргументов и возвращаемого результата. Тип void для пустых функций.
  • День 10: Интерфейсы (interface). Создание жестких шаблонов и паспортов для объектов, защита кода от опечаток в полях данных.
  • День 11: Git — День 1. Регистрация на GitHub, установка Git на ПК. Команды локального сохранения: git init, git add ., git commit.
  • День 12: Git — День 2. Связка локального ПК с облаком GitHub. Команды отправки и скачивания кода: git clone, git push, git pull, создание веток.
  • День 13: Pull Requests. Изучение процесса слияния веток кода на GitHub и механики код-ревью.
  • День 14: Выходной / Закрепление хвостов.

Неделя 3: Тестирование API и Postman

  • День 15: Теория API. Клиент-серверная архитектура, структура HTTP-запроса (URL, методы GET/POST/PUT/DELETE, Headers, Body).
  • День 16: Статусы ответов (200, 201, 400, 401, 500). Установка и обзор интерфейса инструмента Postman.
  • День 17: Практика в Postman. Создание коллекций, отправка запросов к открытым тестовым API, базовая валидация ответов.
  • День 18: Авторизация в API. Передача токенов безопасности (Bearer Tokens, API Keys) в заголовках запросов.
  • День 19: Введение в GraphQL. Ключевые отличия от REST API, написание первых простых запросов query и mutation.
  • День 20: Сквозные сценарии API. Построение цепочки запросов: создание сущности -> проверка данных -> удаление сущности.
  • День 21: Выходной.
